Bug Reporting Process

To report a software bug, installation problem or documentation error in any of our products, please send an e-mail to ProSupport@gemboxsoftware.com with all of the following information:
  1. Full name of the product, together with version information (for example, “GemBox.Spreadsheet Professional 3.5”).
     
  2. Name of the product part affected by the bug (component, installation, help files etc.).
     
  3. Steps to reproduce the bug (together with the code and project files if necessary).
     
  4. Detailed bug description (exception text, stack trace, output files etc.).
     
  5. Your environment information (.NET Framework version, Visual Studio .NET version, operating system etc.).
After our technical department receives your e-mail, they will notify you whether your problem is really a bug, whether it is already a known issue or not, what the workaround is (if available) and when a bug fix will be available. Our technical representative may need to contact you with additional questions to reproduce the bug or to provide workaround in your case.


Feature Request

To request a feature in future version of any of our products, please send an e-mail to ProSupport@gemboxsoftware.com with all of the following information:
  1. Name of the product that should implement the feature.
     
  2. Detailed description of the feature.
     
  3. Reasons why that feature would be valuable for your business.
After our technical department receives your e-mail, they will notify you whether the requested feature is already available in some product version or if that feature is planned in feature versions of the product. Our technical representative may need to contact you with additional questions so we can fully understand what you require and what would be the best way to support it.
